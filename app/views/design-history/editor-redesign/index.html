Problem Statement

Most of Defra's forms were paper-based, making them inaccessible, costly, and inefficient. The existing digital tools available for building forms were unintuitive, inaccessible, and required significant training, limiting the pace and efficiency of digitisation efforts.

This directly impacted:

  • User confidence
  • Form publication speed
  • Widespread adoption by content designers outside the core Defra Forms team
Scope and Aims

Our goal was clear: deliver a timely, cost-effective, accessible, and user-centred digital experience for content designers building online forms. This would streamline digitisation and enable rapid, accurate processing of data submitted by end users.

Initial Assessment

To identify immediate usability issues, we started with heuristic and accessibility testing of the existing service. A senior interaction designer and two senior content designers evaluated the product against NNGroup heuristics and accessibility guidelines.

We also engaged the Defra accessibility team for an independent review, with results pending.

Current State Analysis

We mapped the existing Defra Forms Designer workflows extensively using detailed diagrams in Figma. This allowed us to pinpoint specific areas for improvement and establish baseline metrics for future iterations.

Key findings included:

  • A complex, unintuitive interface
  • Overwhelming information presentation
  • Non-standardised components causing user confusion
Roadmap Reprioritisation

Given significant usability barriers, we decided to reprioritise our product backlog. This allowed our team to focus on re-designing the Forms Designer ahead of developing new features.

Key decisions included:

  • Conducting internal research only on features already in progress (like File Upload and Repeater Additions)
  • Delaying certain feature releases to prioritise usability improvements
  • Targeting delivery of one new feature per month, alongside UX enhancements
Design Process and Improvements

Component Improvements

  • Guided Input Component: Live, dynamic highlighting within a real-time preview panel
  • Preview Panel: Real-time rendering of form edits with improved visual hierarchy
  • Configuration Panel: Structured, accessible layout with progressive enhancement

User Journey Enhancements

We improved the user journey using progressive enhancement principles:

  • Established clear navigation hierarchy
  • Created dedicated primary actions
  • Standardised headings, labels, and action buttons
  • Removed unnecessary UI clutter
Outcomes and Next Steps

Our iterative redesign led to tangible improvements:

  • Users quickly recognised navigation elements and could intuitively interact with the interface
  • Clearer UI reduced cognitive load, improved confidence, and increased task success rates
  • Improved component accessibility and usability, accelerating form creation and publication

Going forward, we plan further research to address remaining user confusion around advanced interactions. Continued iterative development and rigorous user testing will ensure the Forms Designer meets evolving user needs and accessibility standards.
