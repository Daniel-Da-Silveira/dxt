{% extends "design-history/template.html" %} {# --- Feature metadata --- #} {% set featureName = "Preview panel" %} {% set latestVersionLink = "/titan-mvp-1.2/form-editor/question-type/radios-nf/edit" %} {% set status = "Tested" %} {% set statusColor = "green" %} {% set userNeed = "Users need to create and manage different types of list-based questions (radio buttons, checkboxes, autocomplete, yes-no) to collect structured responses from members of the public." %} {% set phase = "Private beta" %} {% set category = ["Form creation", "Members of the public"] %} {% set description %} This feature allows users to create and manage different types of list-based questions in forms, including radio buttons, checkboxes, autocomplete, and yes-no questions. It provides a robust, accessible experience for collecting structured responses.
Key Features and Accessibility
  • Designed for everyone: Uses GOV.UK Design System components, follows form best practice (clear, accessible, forgiving), and includes screen reader support via ARIA live regions.
  • Multiple list types: Supports radio buttons, checkboxes, autocomplete, and yes-no questions with appropriate validation and error handling.
  • Accessible by default: All interactions are accessible via keyboard and screen readers, with clear focus states and error announcements.

Recent improvements:

  • Bulk option entry: Added ability to enter multiple options at once for checkboxes and radio buttons.
  • Improved preview: Enhanced preview panel shows accurate representation of list questions.
  • Better guidance: Clearer explanations of when to use each list type.
  • Question-first approach: Write question text before adding options for better context.
  • Single-page editing: Edit all options on one page for better visibility and control.
Accessibility for Reordering List Items
  • Screen reader announcements: When a user focuses on a "Move" button for an item in the list, the screen reader announces:
    Button, Move [Option text]: [Up or Down], [Option text] is currently option [Position] of [Total options] options.
  • Reordering and announcing the new position: When a user clicks the "Move" button, the list is visually updated and the screen reader announces:
    The list has been reordered, [Option text] is now option [new position] of [Total options] options.
  • Focus retention: After reordering, focus remains on the moved item (and its "Move" buttons) so the user can continue making adjustments without having to refocus.
  • Accessibility and usability requirements: All announcements must include:
    1. What happened: "The list has been reordered..."
    2. Which item was moved: "... [Option text name]..."
    3. The new position: "... is now item [Position] of [Total options] options."
  • Dynamic button labels: Button labels or associated text update dynamically to reflect the new position of the item for any subsequent focus.
  • For more, see: Reordering lists accessibility guidance
Key features and benefits
  • Question-first approach: Write your question before adding options for better context.
  • Bulk option entry: Enter multiple options at once for faster setup.
  • Live preview: See how your list question will appear to users.
  • Clear guidance: Helpful hints about when to use each list type.
  • Accessibility built-in: Screen readers announce options and errors automatically.
  • Error handling: Clear error messages for invalid selections.
What users experience
  1. Choose list type: Select from radio buttons, checkboxes, autocomplete, or yes-no.
  2. Write question: Enter the question text that will appear to users.
  3. Add options: Enter one or more options for the list.
  4. Preview: See how the question will appear to users.
  5. Add guidance: Include helpful text to guide users.
  6. Continue: Save and continue building your form.
