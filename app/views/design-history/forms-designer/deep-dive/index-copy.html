    Editing a Form

    Creating Pages

    Old iteration:

    • confusing relationship between pages and questions
    • limited page type options without clear guidance
    • no preview of how pages would look in the final form
    • technical terminology that confused content designers

    Latest iteration:

    • clear distinction between question pages, guidance pages, and summary pages
    • intuitive page type selection with examples
    • real-time preview of page layout and content
    • plain English terminology that matches content designers' mental models

    "Participants' mental model is to create/add a question to the form." — Research finding from Round 2

    Adding Questions

    Old iteration:

    • difficult to understand different question types
    • no immediate feedback on question setup
    • complex configuration options without guidance
    • separate steps for question text and options

    Latest iteration:

    • clear categorization of question types with examples
    • real-time preview of questions as they're being built
    • guided input with contextual help
    • integrated workflow for question text and options

    "It would be really nice to... think about what's my question text then think about all of my radios at the same time." — Content designer feedback from Round 11

    Adding Conditions

    Old iteration:

    • technical terms like "operators" confused users
    • complex logic building without visual aids
    • conditions separated from question creation
    • no way to preview conditional paths

    Latest iteration:

    • plain English terminology ("condition type" instead of "operator")
    • visual logic builder with clear options
    • integrated condition setup with question creation
    • interactive preview of conditional paths

    "The ability to combine conditions—that's pretty wicked." — Content designer feedback from Round 13

    Re-ordering Content

    Old iteration:

    Latest iteration:

    • intuitive drag-and-drop reordering
    • real-time visual feedback during moves
    • unified interface for all reordering tasks
    • maintains context with preview updates

    Scope and aims of the service

    Our goal was clear: transform the x-gov form builder into a timely, cost-effective, accessible, and user-centred digital experience for content designers building online forms. This would streamline digitisation and enable rapid, accurate processing of data submitted by end users.

    Key objectives included:

    • Achieve WCAG 2.2 compliance across all components and interactions
    • Conform to design standards and best practices
    • Improve intuitiveness and user experience
    • Enable wider adoption by content designers
    • Accelerate form digitisation efforts

    Research and Iterations

    Our redesign process was driven by extensive user research and iterative testing across 13 rounds between May 2024 and April 2025, involving over 50 participants from content designers, proxy users, and internal stakeholders across Defra departments.

    Key Themes and Trends

    • Form Creation Flow: Users preferred a linear, intuitive flow from form name to question creation
    • Page Conditions: Improved terminology from "operator" to "condition type" and better visibility of condition filters
    • Multiple Response Questions: Renamed from "repeater" to "multiple responses" for clarity
    • Visual Design: Positive feedback on GOV.UK style familiarity and white space usage
    • Guidance Tools: Enhanced markdown support and contextual help integration
    • Accessibility: Focus on WCAG 2.2 compliance and inclusive design patterns

    "Even just doing it once made it less confusing the second time." — Content designer feedback from Round 2

    Research Journey

    Our research journey spanned 13 rounds, each building upon previous findings and driving continuous improvements:

    Initial Discovery (Rounds 1-3)

    • Initial research with 6 content designers from various Defra teams
    • Key findings on form library, filters, and terminology clarity
    • Identified need for better search functionality and immediate access to personal forms
    • Clarified confusion around "Published" vs "Live" terminology
    • Component testing showed improved usability with new designs
    • Users responded positively to clearer visual feedback
    • User journey testing confirmed improved navigation and task success rates

    Deep Dive and Refinement (Rounds 4-7)

    • Explored complex interactions like conditional logic and multiple responses
    • Refined terminology and improved guidance clarity
    • Enhanced accessibility features based on user feedback
    • Implemented enhanced component designs with progressive enhancement support

    Validation and Optimization (Rounds 8-10)

    • Tested complete user journeys with diverse user groups
    • Optimized performance and interaction patterns
    • Further refinement of navigation hierarchy implemented

    Final Iterations (Rounds 11-13)

    • Conducted comprehensive accessibility testing
    • Refined edge cases and complex scenarios
    • Validated improvements with new and returning users

    Key Research Outcomes

    • Improved navigation hierarchy and clarity
    • Enhanced component accessibility
    • Streamlined user workflows
    • Better progressive enhancement support
    • Clearer visual feedback
    • Standardised UI patterns
    • Reduced cognitive load
    • Improved task success rates

    "The design feels really clean, really like a lot of space, very methodical step by step" — Content designer feedback

    Component improvements and innovations

    Guided Input Component

    • Live, dynamic highlighting within a real-time preview panel
    • Enhanced visual feedback as content designers input or edit data
    • Clearer placeholders providing immediate context

    Preview Panel

    • Real-time rendering of form edits
    • Clear separation by context (questions, headings, multiple inputs)
    • Improved visual hierarchy and context-aware interactions

    Configuration Panel

    • Structured, accessible layout
    • Clear delineation of settings, options, and actions
    • Progressive enhancement strategies for JavaScript-disabled environments

    "Whatever appears in a green box, I just do it." — Content designer feedback on success messages

    Overall Flow and Page Types

    The Forms Designer follows a clear, intuitive flow that guides content designers through the form creation process. The service is built with progressive enhancement principles, ensuring a quality experience for all users, regardless of JavaScript availability.

    Form Management Pages

    • Form Listing Page: Summary of all created pages and end pages (including Check answers page), with quick preview functionality when JavaScript is enabled
    • Page Re-ordering: Intuitive interface for changing page order using up and down controls
    • Form Preview: Complete preview of the form from start to finish

    Form Creation Pages

    • Page Type Selection: Choose between question or guidance page creation
    • Information Type Selection: Select the specific type of question to add
    • Question Configuration: Configure individual questions with appropriate settings
    • Page Overview: Summary of questions, page heading, guidance, multiple response settings, and question reordering

    Progressive Enhancement Features

    The service is designed to work effectively with or without JavaScript, with specific enhancements for JavaScript-enabled environments:

    JavaScript Disabled

    • Preview panels shown separately from configuration panels
    • Preview buttons for viewing questions/pages or error messages
    • Page reloads required for reordering actions

    JavaScript Enabled

    • Preview panels shown alongside configuration panels
    • Skip links for quick navigation between panels (visible on tab focus)
    • Left-border highlighting for questions in preview panel
    • Guided input with dynamic highlighting of related preview content
    • Smooth reordering without page reloads
    • Focus-based highlighting of corresponding preview elements

    Reordering Features

    The service includes specialized reordering functionality for different elements:

    • Lists (checkboxes, radios)
    • Pages
    • Questions

    All reordering movements include specific screen reader announcements for accessibility.

    Outcomes and next steps

    Our iterative redesign led to tangible improvements:

    • Users quickly recognised navigation elements and could intuitively interact with the interface
    • Clearer UI reduced cognitive load, improved confidence, and increased task success rates
    • Improved component accessibility and usability, accelerating form creation and publication

    Next Steps

    • Continue refining terminology (e.g., 'repeater', 'short description')
    • Improve layout clarity for sections and condition logic
    • Support card sorts and diary studies to enhance guidance IA
    • Conduct accessibility testing and continue inclusive design
