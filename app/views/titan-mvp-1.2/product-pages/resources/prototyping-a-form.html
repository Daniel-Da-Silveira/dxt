Before creating your form, it's useful to first develop a wireframe version of the form. You can do this in Mural. Use one Sticky per question, and map out any routing (for example, for yes/no questions).

Prototyping helps you to:

  • show the user's journey clearly
  • decide where grouping makes sense
  • reduce unnecessary pages
  • avoid redundant questions
  • talk through the form with SMEs and stakeholders

See an example of a form wireframe.

Avoid routing

Routing adds complexity and should be used sparingly.

It's a good idea to:

  • keep users on a single, predictable path
  • reduce the number of decision points

If a route changes the context or focus of the form, consider making multiple context specific forms instead - the data will be more valuable.

Avoid duplicating questions

Reducing duplicate questions improves efficiency and the user experience.

Avoid duplication by:

  • only asking for information once
  • reusing data across different steps
  • allowing users to review and confirm assumed information
  • removing optional fields unless critical
