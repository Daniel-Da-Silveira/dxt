Designing a start page for your form

Use a detailed guide template not a form page in Whitehall.

Form pages add attachments to the top of the page. But ideally, the paper form should go underneath the 'Start Now' button.

If there's an existing form template page for a paper-based form, we recommend unpublishing it and redirecting it to a new detailed guide template.

Here is a example start page for a form using the detailed guide template.

Including a start page button on an existing guidance page

You can add a 'Start now' button to an existing guidance page on GOV.UK for the form you're working on.

Start page buttons should only appear once and be used at the start of your form.

Use Govspeak (a simplified version of Markdown) to format the button.

Consider the position of the 'Start now' button and how it might change if the content is amended. Get sign off of the start page early on so that it hold things up when the form is otherwise ready to be published.

Where appropriate, speak to other content designers who may have recently updated the content, to find out about upcoming amends or improvements. See Content design guidance for more information.

Start page layout

We recommend using a clear structure for your form start page. Say who should use the form in the page summary. Here's an example from a live form.

"Fire and rescue services in England can use this service to request a pump service or environmental protection equipment from the Environment Agency."

The form start page should also include:

  • a form introduction including links to any related guidance
  • a before you start section that lists anything users will need like a reference number or documents before they start
  • an 'Other ways to apply' section if there's a paper form
  • a 'Get help' section where we add the support details
  • terms and conditions section if needed

Use this template if you're sending a form by email instead of hosting on a GOV.UK start page.

What form guidance should go on a GOV.UK page

Guidance for filling in the form should be built into the form, not separate to the form. Do not have a separate 'how to fill in this form guide' document. It's hard for users to refer back and forward between tabs or documents. See Service manual: structuring forms for more information.

Guidance to be read before completing the form must be listed in a before you start section. Avoid referring people out of the form.

Guidance for each section must be in that section.

Do not link to guidance that could be added to the form - for example a couple of sentences within a guide on GOV.UK. Instead, extract the text and add it to the form.

Do not put guidance text next to a question that is not related to the actual question.
