5. Where team priorities lie and which improvements will deliver the most value

The team uses evidence-based decision making to prioritise improvements, focusing on changes that deliver the most value to users and the service.

Example 1: Prioritising Complete Redesign Based on User Impact

Situation:
Initial evaluation (March 2024) revealed significant usability issues. The team needed to decide whether to make incremental improvements or prioritise a complete redesign.

Task:
Determine which approach would deliver the most value: incremental improvements or complete redesign.

Action:

  • Evaluated severity of usability issues and user impact
  • Assessed that incremental improvements would not address fundamental problems
  • Prioritised complete redesign to address root causes
  • Reprioritised product backlog to focus on redesign
  • Transformed from component-driven to task-driven approach

Result:

  • Complete redesign validated through 13 rounds of research
  • Users consistently report service as "intuitive", "clean", and "methodical step by step"
  • Demonstrated prioritisation based on user impact rather than ease of implementation
  • Long-term value delivered through addressing root causes

Example 2: Prioritising Quick Wins Alongside Major Improvements

Situation:
Multiple issues identified in research rounds. Team needed to balance quick wins with longer-term improvements.

Task:
Prioritise improvements to deliver value quickly while working on larger changes.

Action:

  • Identified quick wins: terminology improvements, discoverability fixes
  • Implemented rapid changes (e.g., "operator" to "Condition type")
  • Continued work on larger improvements (e.g., question creation flow)
  • Balanced immediate user value with longer-term improvements

Result:

  • Users saw immediate improvements from quick wins
  • Larger improvements delivered over time
  • Demonstrated ability to prioritise based on value and feasibility
  • Continuous improvement maintained even with limited resources

Example 3: Using Data to Prioritise Improvements

Situation:
User feedback about "save and exit" feature indicated issues, but scope was unclear.

Task:
Use data alongside feedback to understand priority and value of improvements.

Action:

  • Analysed usage data to understand scope of issue
  • Found 96% success rate - feature works well for most users
  • When it fails, impact is significant for affected users
  • Used data to inform discussion about improvement priorities
  • Considered multiple improvement options with different effort/impact ratios

Result:

  • Data-informed prioritisation rather than assumptions
  • Understanding of where improvements would deliver most value
  • Multiple options considered with different effort/impact trade-offs
  • Demonstrated evidence-based approach to prioritisation

Example 4: Prioritising Features Based on User Needs

Situation:
Multiple features could be developed. Team needed to prioritise based on user value.

Task:
Prioritise feature development based on user needs and value delivered.

Action:

  • 27 features (84%) tested and validated with users
  • Features prioritised based on user research findings
  • User needs drive prioritisation rather than technical ease
  • Average 2.4 iterations per feature to ensure quality

Result:

  • Features developed based on user value
  • High percentage of features tested ensures quality
  • Multiple iterations ensure features meet user needs
  • Prioritisation aligned with user needs and service goals
